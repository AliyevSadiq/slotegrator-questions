Стоит задача реализовать возможность работы с билетами
которые хранятся на другом сервере (по API).
Требуется описать структуру методов и классов, с помощью
которых можно будет загружать билеты как из БД, так и из
другого сервера (по API).


--------------------------------------------------------------------------

    1. Для решения задачи в среде "Laravel" мы можем создать интерфейс "TicketRepositoryInterface"
      и добавить сигнатуры методов, которые нам нужны

    2.1. Затем создать два класса репозитория с именами "ApiTicketRepositoryRepository"
      и "DbTicketRepositoryRepository" и реализовать этот интерфейс для этих классов.
    2.2. Реализовать методы интерфейса в этих классах


    3.1 Для использования выбранного класса репозитория по интерфейсу мы можем использовать
      внедрение зависимостей, для этого мы создаем поставщика услуг с именем "TicketServiceProvider"
      и реализуем внедрение зависимостей в методе загрузки "TicketServiceProvider".
    3.2. После этого добавьте "TicketServiceProvider" в конфигурационный файл "app.php" в массив "providers".


    4.1. Кроме того, мы должны добавить в файл ".env" свойство "TICKET_REPO"
    4.2. И мы должны добавить в файл конфигурации "services.php" свойство "ticket_repo"

    5.1. И последнее для работы с выбранным репозиторием используем интерфейс "TicketRepositoryInterface"
      в конструкторе или как аргумент метода
